## A. Host flow — make it linear (the big theme)

The host page exposes the whole state machine flat ("asking too much of
whoever's running the game"; "I can go from draft to tiebreak — that doesn't
make any sense"). Replace with a guided, linear flow:

1. **A-1** Official question cutoff: one action that closes submissions — the
   moment the game "starts". No more contributions after.
2. **A-2** Then facilitator picks MC mode: Lacey in Charge (default
   expectation) or Gladys.
3. **A-3** Then rounds are built automatically under hard rules (see B).
4. **A-4** From then on, a single "next" progression: open round → pens down →
   mark → reveal → next round → … → final wager → final → (tiebreak) → done.
   Free-jump phase buttons go away (or hide behind an "advanced" escape hatch).

## B. Round building — hard rules

- **B-1** Everyone who submitted gets ≥1 of their questions in the game.
  Guaranteed.
- **B-2** Rounds are even; **max 5 questions per round**; **every category is
  a round** (5 per round is the target shape).
- **B-3** New setting, adjustable during the gathering phase: **questions kept
  per person (N=1..5)**. Keep the FIRST N in the order the person submitted
  them. (Set to 1 → keep only their first question; 5 → keep all five, in
  order.) Non-destructive: extra questions stay stored, just not selected.
- **B-4** Open design question (needs Travis): with 9 categories, thin
  categories won't fill 5 slots. Proposal: only categories with enough
  questions become rounds; leftovers merge into a "Mixed Bag" round; bank
  questions (see C) fill gaps.

## C. Question bank

- **C-1** Research task: find a good bar-trivia-style question bank,
  actively maintained (updated within ~the last 3 months — recency of
  maintenance, not recency of topics). Search reddit/subreddits etc.
  Check license before importing.
- **C-2** Host can add questions to the bank AS HOST (not as a player);
  hers get pulled into the per-category pools like everyone else's.
- **C-3** Question bank management UI for the host: see ALL questions in a
  simple table/grid, laptop-focused. Simplicity over polish.
- **C-4** Final round: host should NOT have to author it. Offer a curated
  pick-list of final-round questions (answers hidden from the picker so the
  setup person can still play). Same for **tiebreak questions** — a good
  pick-list.

## D. Team / join UX (player surface)

- **D-1 BUG** After "Join the game": toast fired but the UI stayed on the
  pick-a-team-name page. Should navigate to the team/lobby view immediately.
- **D-2 BUG?** Leaving and returning to /play asked him to pick a team again —
  localStorage team memory either didn't persist or didn't rehydrate.
  Investigate (join handler may not persist team_id, or applyState raced).
- **D-3** KILL recovery codes ("a very bad idea"). No security wanted at all.
  Rejoining = show the list of existing teams, tap one to hop on (anyone can).
  Creating another team from that list is fine too.
- **D-4** DROP individual team members entirely. Submissions are BY TEAM;
  we don't need to know who's on a team. Remove the team-builder/roster from
  the player flow (kills a whole feature — good).
- **D-5** Display screen showed the new team in real time — that part is right.

## E. Marking — Lacey console

- **E-1** Photo-snap stays the preferred confirmation path — but it must be
  performant. Travis will test it; if unwieldy, fall back to E-2 as primary.
- **E-2** Lacey grid: show her the answer + each team's response, she clicks
  right/wrong per team per question; ALL scoring happens automatically —
  she never thinks about points.
- **E-3** Token burn is fine: AI can pre-process (transcribe/propose) and
  Lacey verifies on screen. (This is Gladys-assist inside Lacey mode —
  photo → AI proposal → human confirm, without giving up manual control.)
